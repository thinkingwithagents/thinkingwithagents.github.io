Outside Resources

Tools, guides, and examples beyond the bootcamp itself

This page is a broader reference list for people who want to keep going after the bootcamp. It is organized by resource type so you can jump directly to product docs, teaching guidance, research tools, or workflow examples.

How to use this page

Pick one section that matches what you need right now:

  • official documentation if you want to learn a tool directly
  • teaching guidance if you want faculty-oriented pedagogy examples
  • research and literature tools if you want help with discovery or evidence synthesis
  • workflow examples if you want to see how other people actually use these tools
  • notes and productivity tools if your bottleneck is organization rather than generation

You do not need to read everything here.

Official tool documentation

UVM and Microsoft

  • Microsoft Copilot at UVM Best starting point if you want the UVM-supported option.
  • Microsoft Copilot docs Useful if you want the broader Microsoft ecosystem view.

Anthropic / Claude

  • Claude Code docs Best starting point for the terminal-based, file-aware workflow featured in the bootcamp.
  • Anthropic prompt engineering guide Useful for improving prompting and workflow structure.

OpenAI / ChatGPT / Codex

  • ChatGPT Main chat-first entry point for ChatGPT.

  • OpenAI Codex CLI getting started Best if you want the terminal workflow specifically.
  • OpenAI ChatGPT Education docs Useful if you want educator-oriented OpenAI guidance.

Alternative editor-based workflow

  • Cursor Popular option for people who want an editor-based AI workflow rather than a terminal-first workflow.
  • Cursor docs Useful if you want to compare editor-based and terminal-based workflows.

Teaching and pedagogy guidance

Higher-ed teaching guidance

  • Columbia CTL: Teaching and Learning in the Age of AI Strong higher-ed resource for faculty trying to think through teaching, assignments, and policy.
  • Columbia CTL: Faculty use cases Concrete examples of how faculty across disciplines are using AI.

Teaching use cases to try

These are the teaching tasks where the tools above tend to be most useful:

  • drafting assignment prompts
  • revising instructions for clarity
  • building rubrics
  • generating study guides
  • creating practice questions
  • scaffolding feedback comments

If you want a first experiment, try one bounded teaching task using an existing course document rather than starting from a blank page.

Research and literature tools

Literature review and evidence synthesis

  • Elicit Good for literature review, evidence synthesis, and research reports.
  • Scite Especially useful for citation context and seeing whether later work supports or contrasts with a claim.

Literature discovery and mapping

  • ResearchRabbit Useful for citation-network exploration and discovering related work.

Research tasks where AI is often useful

  • literature review organization
  • coding assistance
  • data cleaning and exploration
  • first-pass tables, graphs, and memos
  • writing and revision
  • project handoff notes and reproducibility checklists

The safer pattern for research is usually:

  • keep the task narrow
  • save intermediate outputs
  • review before building on them
  • avoid exposing protected raw data unless you have a clear reason and safe setup

Workflow examples and explainers

These are helpful when official documentation feels too product-focused and you want to see how a real person uses the tools.

Paul Goldsmith-Pinkham

  • Getting started with Claude Code A good introduction to why terminal-based AI work feels different from ordinary chat.

Claude Blattman

  • Claude Blattman Useful for concrete workflows, reusable patterns, and examples aimed at professionals who are not primarily programmers.

Notes, Markdown, and knowledge management

These are useful because AI tools often produce Markdown files, memos, and reusable prompts.

Markdown viewers and editors

  • Typora A good default Markdown viewer if you want AI-generated .md files to look like normal formatted text immediately.
  • Zettlr Worth a look if you want a more writing-oriented Markdown environment.

Notes and project memory

  • Obsidian Useful for storing prompts, session notes, project instructions, and reusable workflows.
  • Obsidian Help Good if you want to learn links, templates, vault organization, and lightweight knowledge management.

Meeting capture and dictation tools

These are optional, but useful if your work starts from spoken notes, meetings, or rough fragments.

  • Granola Useful for meeting capture, searchable notes, and turning conversation into a more usable record.
  • Wispr Flow Useful for dictation-heavy workflows and quick drafting on computer or phone.

Workflow ideas worth exploring later

If the basic tools feel useful, the next level is not just better prompting. It is better workflow structure:

  • reusable prompts
  • project instruction files such as CLAUDE.md or AGENTS.md
  • saved Markdown outputs
  • task plans and handoff notes
  • comparing two outputs instead of trusting the first one
