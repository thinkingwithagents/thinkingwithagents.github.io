Recap: The Compound Workflow (Emily)

~10 minutes

Last session: one prompt → one output. Today: chained steps where AI builds on its own prior work.

The cycle

Plan → Execute → Review → Clear context → Repeat

Why clear context? AI tools have limited memory. A fresh conversation with a clear task beats a long, muddled conversation. Document what you learned in a markdown file so you (and the AI) can pick up where you left off.

Key principles

  • Break big tasks into steps: Don’t ask “analyze my data and write a paper.” Ask “read my data dictionary and propose an analysis plan.”
  • Save intermediate outputs: Write results to files, not just chat.
  • Review each step: AI makes confident-sounding mistakes. Check before building on top.
  • Document as you go: Keep a running markdown file of decisions, prompts that worked, and lessons learned.

Part 1: Research Pipeline Demo (Emily)

~35 minutes

Live demo: Raw data → analysis plan → code → report

Starting from a raw dataset, we’ll use Claude Code to:

  1. Explore the data: “Read this CSV and describe what’s in it. How many observations? What are the variables?”
  2. Propose an analysis plan: “Given these variables, suggest 3 analyses that would be interesting for a research paper on [topic].”
  3. Write analysis code: “Write Stata code to run [analysis]. Use clear variable labels and save the output.”
  4. Generate a summary report: “Based on these results, draft a 1-page summary with key findings, a table, and a limitations paragraph.”
NoteWhat this demo is NOT

This is not a replacement for careful research design. The AI doesn’t know your research question, institutional context, or identification strategy. It’s a tool for accelerating the mechanical parts — data cleaning, boilerplate code, first-draft summaries — so you can focus on the thinking.
