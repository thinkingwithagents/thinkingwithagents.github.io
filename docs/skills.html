Skills & Demos

Reusable AI workflows you can install and try

One of the most useful things about agentic AI tools is that workflows can be packaged as skills — reusable prompt-and-instruction bundles that anyone can install and run. Think of them as saved recipes for tasks you do repeatedly.

This page collects skills and demos from the bootcamp. You can use them as-is, adapt them, or just read them to see how structured AI workflows are built.

What is a skill?

A skill is a set of instructions — usually a Markdown file — that tells an AI tool how to do a specific task. When you invoke a skill, it loads those instructions automatically.

For example, instead of explaining “read my draft, check the causal claims, flag anything that’s correlational language dressed up as identification” every time, you write that once and save it as a skill. Then you just type /econ-audit and it does it.

Skills are plain text files. You can read them, edit them, and share them.

How to install a skill

NoteClaude Code (terminal)

Skills can be installed from skills.sh with one command:

claude skills add <skill-name>

Or you can drop a .md file into your .claude/commands/ folder to create a local skill.

NoteOther tools

Most agentic tools support some form of reusable instructions. In Cursor, these go in .cursorrules. In Codex, use AGENTS.md. The concept transfers even if the file format differs.

TipInstall the bootcamp skills

These skills are published and ready to use:

npx skills add eabeam/econ-skills

Or install one at a time:

npx skills add eabeam/econ-skills --skill econ-audit

Browse the source code: github.com/eabeam/econ-skills

Skills from this bootcamp

These are real skills built for economics research and teaching workflows. They are not toy examples.

Literature Review (/lit-review)

Structured literature search and synthesis. Give it a research question, and it searches for relevant papers, organizes them by theme, identifies gaps, and produces a formatted review memo.

Good for: starting a new project, preparing a referee report, catching up on a subfield.

Install: npx skills add eabeam/econ-skills --skill lit-review | Browse source

Econometrics Audit (/econ-audit)

Adversarial review of empirical claims. It reads your draft or code and flags: causal language without identification, missing robustness checks, specification concerns, and measurement issues.

Good for: self-review before submission, preparing for seminars, reviewing student work.

Install: npx skills add eabeam/econ-skills --skill econ-audit | Browse source

Data Dictionary Generator (/data-dictionary)

Reads a dataset’s metadata and produces a formatted data dictionary with variable names, types, labels, value ranges, and notes. Saves time on replication package documentation.

Good for: replication packages, project handoffs, codebook creation.

Install: npx skills add eabeam/econ-skills --skill data-dictionary | Browse source

Building your own

The best skill is one you build for a task you actually do. Start with:

  1. Pick one repeating task — grading feedback, email drafts, data cleaning, meeting prep
  2. Do it once with AI and save the conversation
  3. Extract the instructions into a Markdown file
  4. Refine — add constraints, examples, edge cases as you use it

Your instructions file doesn’t need to be perfect. It just needs to be better than explaining the task from scratch every time.
