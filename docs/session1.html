Session 1: Getting Started with AI Tools

April 22, 2026 — AI Bootcamp for UVM Economics

Overview

Time: 1.5 hours (lunch session) Presenters: Emily Beam + Erkmen Aslim Goal: Get faculty comfortable using AI tools for everyday academic tasks. By the end, everyone should have used an AI tool to produce something useful.

The AI Ladder

Where are you today?

Level Description Example
0 Haven’t tried it —
1 Tried it once, wasn’t impressed Asked ChatGPT a question, got a bad answer
2 Use it occasionally for simple tasks Drafting emails, brainstorming
3 Regular user, integrated into workflow Writing assistance, code help, literature search
4 Power user, multi-step workflows Research pipelines, automated grading, custom tools
5 Building systems with AI Custom agents, teaching tools, research infrastructure

Our goal today: Get everyone to at least Level 2, and show you what Levels 3–4 look like.

NoteFormat

This is a watch-along session — we’ll demo everything live. No installation required. Slides will be up during generation/loading time so you can follow the concepts while the AI works.

Part 1: Terminal + Claude Code (Emily)

~30 minutes

Why the terminal?

The desktop chat interfaces (ChatGPT, Claude.ai) are great for quick questions. But the terminal unlocks:

  • File access: AI can read and write files on your computer
  • Iteration: Work on a document across multiple steps without copy-pasting
  • Reproducibility: Save your workflow, rerun it later
  • Power tools: Claude Code can search your files, run code, and build things

What you’re seeing

  1. Ghostty — a terminal app (any terminal works; this one just looks nice)
  2. Claude Code — an AI assistant that runs in the terminal. Type claude to launch.
  3. Live prompting — we type a request, the AI responds, we iterate.
TipThings you could try later on your own
  • “Read my syllabus at [path] and list all the due dates in a table”
  • “I have a CSV of student grades. Summarize the distribution and flag any outliers.”
  • “Draft a response to this student email: [paste email]”

Key concepts

  • Context window: The AI can only “see” what you show it. Be explicit about what you want.
  • Iteration: Your first output won’t be perfect. Refine: “Make it shorter,” “Add a section on X,” “Change the tone to be more formal.”
  • Markdown output: Claude Code writes in Markdown. Use Typora to view it nicely.
